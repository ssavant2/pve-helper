Shared storage gate

This gate protects orphan classification and destructive file operations; it is not the storage's availability status. If an expected Proxmox consumer is offline, browsing through PVE-helper and guest operations on healthy nodes remain available, while risky file changes stay disabled until inventory is complete. Local/API-only storages are not part of this gate.

A destructive file action must pass two independent checks, because there are two separate ways to be missing evidence. This gate proves that PVE-helper's own scan saw the whole picture: every expected Proxmox consumer of this shared mount answered while the scan ran. The volume coverage shown per datastore in Storage catalog proves that Proxmox's API listing is complete for that exact storage scope, bound to the generation it was observed under. Neither implies the other, so both must pass — and either one refusing is reported with its own reason.
