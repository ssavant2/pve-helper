What do these mean?
  • Defaults are the most compatible. SATA disk, E1000 NIC and SeaBIOS + i440fx boot almost any image. If you know how the image was built, pick matching values.
  • Bootable disk. The selected image must be the guest's bootable OS disk — a data-only disk will attach but won't boot.
  • VirtIO needs drivers. SCSI/VirtIO disk and the VirtIO NIC are faster but require the guest to already have VirtIO drivers (Linux usually does; Windows often doesn't).
  • UEFI images need BIOS = OVMF. A fresh EFI vars disk with Secure Boot keys is created automatically; point to your own EFI vars file only if the image needs it.
  • Chipset. i440fx is the most compatible and boots almost any image; q35 (modern, PCIe) is what many newer OS images expect.
  • If it doesn't boot, it's usually a BIOS / machine / disk-bus mismatch — change those and reboot, no re-import needed.
